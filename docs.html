PRS Personal Recording System Open /prs/ →
Perception · capture & retrieve

Your personal archive: capture anything worth remembering — an idea, a decision, a task, an observation — then find it again through search, timelines, and graphs.

Capture a record

  1. Press N (or click New Record) for the full editor, or Ctrl+⇧+N for quick capture.
  2. In the editor, enter Content (required), then choose Type, Status, Priority, Domain, and add Tags. Save.
  3. In quick capture, the first line becomes the title. Prefix a line to auto-classify it: TODO→Task, DECISION→Decision, GOAL→Goal, ?→Question; otherwise it becomes an Idea.

Find things

  1. Use the header Search — it matches content, notes, tags, subject, domain, and type.
  2. Browse by view from the sidebar: Dashboard, All Records, Timeline, Activity Heatmap, Relationship Graph, or By Type.
  3. Filter All Records by Status, Priority, or Type with the dropdowns.

Working memory

Press Ctrl+⇧+W to open a scratchpad of todos and notes. Click Generate Log Record to fold what you did into an Observation record.

PKTS Personal Keyword Tracking System Open /pkts/ →
Audit · read-only telemetry

A telemetry lab: it analyzes a dataset of keystroke events to surface your typing rhythm, cognitive-load signals, and workflow patterns.

Read the analysis

  1. Land on Overview for volume, speed, consistency, and fatigue at a glance.
  2. Work through the tabs: Behavioral Signature (your typing fingerprint & ergonomics), Temporal Dynamics (timing structure), Composition Process (how you draft), Cognitive State (load, fatigue, flow), Workflow & Expertise (editing-state mining), and Raw Telemetry (the source rows).
  3. On Raw Telemetry, use the filter box to narrow events by key, session, or key code.

PAIS Personal Application Interaction System Open /pais/ →
Audit · interaction telemetry

A sibling of PKTS. It records mouse events (clicks, scroll, sampled movement) and the focused-window timeline, then joins them with the PKTS keystroke stream to surface where your time and input actually go — per application.

Read the dashboard

  1. Start at the Totals row: active time, keystrokes (from PKTS), clicks, scrolls, app switches, and keys-per-click.
  2. The Time-per-application chart and Application breakdown table show how time, input, and attention-fragmentation distribute across apps.
  3. The Activity timeline plots keystrokes / clicks / scrolls in 5-minute buckets; Idle gaps lists periods of no input.
Shared focus watcher PAIS and PKTS share shared/focus_watcher.py as the single source of truth for the focused window, so every keystroke and mouse event carries identical application attribution. On stock GNOME Wayland attribution degrades to unknown (Introspect is restricted); capture continues regardless.
PTOCS Personal Technical Object Catalog System Open /ptocs/ →
Intelligence · catalog & analyze

A catalog of everything you rely on to work — software, services, hardware, references, capabilities — with search, faceted browsing, a relationship graph, and a statistical overlay that flags gaps and risks.

Add an object

  1. Press N (or New Entry).
  2. Enter Name and pick an Object Kind — e.g. software_tool, service_platform, infrastructure, hardware_device, reference_artifact, capability_skill.
  3. Fill Category, Domain, Status, Summary, and vendor/cost as relevant. Save.

Explore

  1. Open Browse to pivot by kind, domain, status, hosting, and more; click any facet to drill in.
  2. Open the Relationship Graph to see how objects connect.
  3. Open Statistical Overlay for findings: single points of failure, redundancy, stale entries, vendor concentration, capability gaps, and cost exposure.

Maintain

Open any entry to Edit, Pin, Retire, or Delete it, or append to its annotation log.

PPS Personal Policy System Open /pps/ →
Policy · read & search

Your policy corpus: a charter, principles, values, commitments, and domain policies (health, learning, conduct) — browsed and searched from one page.

Read

  1. The Charter is pinned as the apex document — start there for identity and purpose.
  2. All other policy cards are grouped by domain below it. Click any card to read it.

Search

  1. Click the search box (or press /).
  2. Type a word; matches appear with highlighted snippets. It's a simple word search — no boolean or field syntax.

AOOS Agent Operation Organization System Open /aoos/ →
Operations · plan, schedule, sync

Organize your work: action constructs (objectives → projects → tasks), a calendar with conflict detection, Google Calendar sync, and a performance scorecard.

Plan your work

  1. Press N (in Actions) to create an action. Pick a Kind — Objective, Initiative, Project, Task, Routine, or Commitment.
  2. Set Effort, Confidence, and the Capacity it draws on; slot it under an Objective / Initiative / Project.
  3. Open Action Hierarchy to see the tree, or filter Actions by Kind, scheduling state, pinned, or blocked.

Schedule it

  1. Open Calendar and choose Day / Week / Month / Year.
  2. Click Schedule Block, pick an action (Task, Routine, or Commitment), set start/end times, and save. Overlapping blocks are flagged as conflicts.

Track time (work sessions)

  1. Use the header focus bar: type what you're working on and click Start (or press Space). The timer runs live; click Stop (or Space again) and add a closing note. An action link is optional.
  2. Open Sessions to see every session grouped by day, with totals. Filter by action or status, edit any entry, or Add manually to backfill a session.
  3. Only one session runs at a time — starting a new one stops the previous (with a prompt). Sessions are the actual counterpart to planned blocks; the gap between them is the feedback signal.

Sync Google Calendar

  1. Open the Google Calendar view and click Connect; complete Google's consent screen.
  2. Click Sync to pull remote events and push your blocks. Without Google credentials it runs in mock mode (a no-op).

Read your performance

Open Analytics for the performance scorecard and roughly twenty indicators across flow, backlog, temporal rhythm, and risk.

PRAS Personal Reflection & Adaptation System Open /pras/ →
Feedback · reflect & adapt

A corpus of deliberations — the structured reflections that observe what happened, make sense of the gap between intention and outcome, and mature into adaptations that feed policy and the rest of the loop.

Read & browse

  1. The Reflection Practice is pinned as the apex document — start there for the method that governs every deliberation.
  2. Below it, all deliberations are grouped by domain, each shown with its type, status, and date. Click any card to read it.
  3. Narrow the index with the Status (open / concluded / enacted / superseded) and Type (review / deviation / retrospective / hypothesis / decision-in-formation) filters.

Search

  1. Click the search box (or press /).
  2. Type a word; matches appear with highlighted snippets, ranked by field. Simple word search — no boolean or field syntax.

The deliberation arc & lifecycle

Each deliberation follows one movement: observation → deliberation → adaptation. It matures along open → concluded → enacted (or superseded). The concluded → enacted step is exactly where a reflection feeds the Policy System and the rest of the loop — the adaptation declares its destination (PPS, AOOS, PRS, PTOCS).

Honest Status

Work in Progress

Autoregia is a set of working prototypes of a larger control loop. Each tool stands on its own, but they are not yet wired together: no tool calls another, and the loop does not yet close. Treat the per-tool notes above as the source of truth for what works today.

For the technical docs Conceptual foundations, data models, and schemas live in the spec/ directory. See spec/README.md for the index, and Closing the Loop for the roadmap that connects these prototypes into one self-regulating system.
