## Version Name Scheme

>  Use the `Major.Minor.Patch` Model.

### Principles

- **Major**: Increment when introducing **breaking changes** or significant redesigns.
- **Minor**: Increment when adding **new features** that are backward-compatible.
- **Patch**: Increment for **bug fixes**, performance improvements, or small non-breaking changes.


### Versioning Grammar

`[Major].[Minor].[Patch]`

 Where:

- **Major** ∈ {1, 2, 3, ..., 5}  
- **Minor** ∈ ℕ₀ {0,1, 2, 3, ..., 10}  
- **Patch** ∈ ℕ₀ {0,1, 2, 3, ..., 15}
