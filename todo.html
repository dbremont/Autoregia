Closing the Loop

A map of the work needed to turn five decoupled prototypes into one self-regulating control loop — the concrete Personal Viable System Model.

Where We Stand

The PVSM defines a closed loop — perceive → estimate → decide → select → execute → feed back. Autoregia has five working prototypes for the first four stages, but they are structurally decoupled islands: no system calls another, there is no coordination substrate, no regulator, and the Execution & Feedback stages have no code. The loop is, today, a diagram. This page maps what stands between the diagram and a running closed loop.

the closed loop today: four amber islands, two red gaps, no substrate · every arrow is currently broken
The read on the diagram: the prototypes are real but inert — none emits a signal another can hear. The single highest-leverage missing piece is the PEB event bus (the substrate band): without it, no arrow on this diagram can carry data. Build the substrate first; the islands can then be wired one at a time, each a small, independently testable step.

Suggested Path

A Pragmatic Sequence

The dependencies point one way. A minimal viable loop — the smallest thing that genuinely self-regulates — needs only: A1 (schema) → A2 (bus) → B1+B5 (PRS emits, PWOS listens) → C2 (regulator reading the existing analytics) → D2 (feed deviations back into PWOS). That five-step spine closes the loop; everything else enriches it.

Definition of done — the loop is closed when: a commitment recorded in PRS appears as an action in PWOS without manual transfer; a deadline missed in PWOS moves the performance scorecard; and the scorecard's deviation produces a rescheduled corrective block — all through the bus, with no human copy-paste between systems.
